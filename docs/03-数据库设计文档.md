### 关系总结

| 关系 | 说明 |
|------|------|
| users 1:N conferences | 一个用户（组织者）可创建多个会议 |
| users 1:N attendees | 一个用户（参会者）可参加多个会议 |
| conferences 1:N invite_codes | 一个会议可生成多个邀请码 |
| conferences 1:N attendees | 一个会议可有多个参会者 |
| attendees 1:N payments | 一个参会记录可有多笔缴费 |
| attendees 1:1 checkins | 一个参会记录对应一次签到 |
| attendees 1:N accommodations | 一个参会记录可有多个住宿记录（换房场景） |
| users 1:N checkins | 一个组织者可为多人签到 |

## 2. 表结构详细设计

### 2.1 users 表（用户）

| 字段 | 类型 | 约束 | 默认值 | 说明 |
|------|------|------|--------|------|
| `id` | INT | PK, AUTO_INCREMENT | — | 用户ID |
| `username` | VARCHAR(50) | UNIQUE, NOT NULL | — | 用户名 |
| `password` | VARCHAR(64) | NOT NULL | — | 密码（MD5+SALT加密后的值） |
| `role` | ENUM('attendee','organizer','admin') | NOT NULL | — | 用户角色 |
| `phone` | VARCHAR(20) | — | NULL | 手机号 |
| `email` | VARCHAR(100) | — | NULL | 电子邮箱 |
| `status` | TINYINT | NOT NULL | 1 | 状态：1正常，0禁用 |
| `created_at` | DATETIME | NOT NULL | CURRENT_TIMESTAMP | 注册时间 |

**索引**：
- PRIMARY KEY (`id`)
- UNIQUE INDEX `idx_username` (`username`)

### 2.2 conferences 表（会议）

| 字段 | 类型 | 约束 | 默认值 | 说明 |
|------|------|------|--------|------|
| `id` | INT | PK, AUTO_INCREMENT | — | 会议ID |
| `organizer_id` | INT | FK → users.id, NOT NULL | — | 组织者ID |
| `title` | VARCHAR(200) | NOT NULL | — | 会议标题 |
| `description` | TEXT | — | NULL | 会议描述 |
| `venue` | VARCHAR(200) | — | NULL | 会议地点 |
| `start_date` | DATE | — | NULL | 开始日期 |
| `end_date` | DATE | — | NULL | 结束日期 |
| `status` | ENUM('pending','approved','rejected') | NOT NULL | 'pending' | 审核状态 |
| `created_at` | DATETIME | NOT NULL | CURRENT_TIMESTAMP | 创建时间 |

**索引**：
- PRIMARY KEY (`id`)
- INDEX `idx_organizer` (`organizer_id`)
- INDEX `idx_status` (`status`)

### 2.3 invite_codes 表（特邀码）

| 字段 | 类型 | 约束 | 默认值 | 说明 |

### 2.4 attendees 表（参会记录）

| 字段 | 类型 | 约束 | 默认值 | 说明 |

## 3. 建表 SQL

```sql
-- 创建数据库
CREATE DATABASE IF NOT EXISTS meeting_db DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
USE meeting_db;

-- 1. 用户表
CREATE TABLE users (
    id INT AUTO_INCREMENT PRIMARY KEY,
    username VARCHAR(50) NOT NULL UNIQUE,
    password VARCHAR(64) NOT NULL,
    role ENUM('attendee','organizer','admin') NOT NULL,
    phone VARCHAR(20),
    email VARCHAR(100),
    status TINYINT NOT NULL DEFAULT 1,
    created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP
) ENGINE=InnoDB;

-- 2. 会议表
CREATE TABLE conferences (
    id INT AUTO_INCREMENT PRIMARY KEY,
    organizer_id INT NOT NULL,
    title VARCHAR(200) NOT NULL,
    description TEXT,
    venue VARCHAR(200),
    start_date DATE,
    end_date DATE,
    status ENUM('pending','approved','rejected') NOT NULL DEFAULT 'pending',
    created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (organizer_id) REFERENCES users(id) ON DELETE CASCADE
) ENGINE=InnoDB;

-- 3. 特邀码表
CREATE TABLE invite_codes (
    id INT AUTO_INCREMENT PRIMARY KEY,
    conference_id INT NOT NULL,
    code VARCHAR(20) NOT NULL UNIQUE,
    max_uses INT NOT NULL DEFAULT 1,
    used_count INT NOT NULL DEFAULT 0,
    created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (conference_id) REFERENCES conferences(id) ON DELETE CASCADE
) ENGINE=InnoDB;

-- 4. 参会记录表
CREATE TABLE attendees (
    id INT AUTO_INCREMENT PRIMARY KEY,
    user_id INT NOT NULL,
    conference_id INT NOT NULL,
    arrival_time DATETIME,
    departure_time DATETIME,
    accommodation_type VARCHAR(50),
    requirements TEXT,
    status TINYINT NOT NULL DEFAULT 1,
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE,
    FOREIGN KEY (conference_id) REFERENCES conferences(id) ON DELETE CASCADE,
    UNIQUE INDEX idx_user_conference (user_id, conference_id)
) ENGINE=InnoDB;

-- 5. 缴费记录表
CREATE TABLE payments (
    id INT AUTO_INCREMENT PRIMARY KEY,
    attendee_id INT NOT NULL,
    amount DECIMAL(10,2) NOT NULL,
    status ENUM('unpaid','paid') NOT NULL DEFAULT 'unpaid',
    paid_at DATETIME,
    FOREIGN KEY (attendee_id) REFERENCES attendees(id) ON DELETE CASCADE
) ENGINE=InnoDB;

-- 6. 签到记录表
CREATE TABLE checkins (
    id INT AUTO_INCREMENT PRIMARY KEY,
    attendee_id INT NOT NULL UNIQUE,
    checkin_time DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    checked_by INT NOT NULL,
    FOREIGN KEY (attendee_id) REFERENCES attendees(id) ON DELETE CASCADE,
    FOREIGN KEY (checked_by) REFERENCES users(id)
) ENGINE=InnoDB;

-- 7. 住宿记录表
CREATE TABLE accommodations (
    id INT AUTO_INCREMENT PRIMARY KEY,
    attendee_id INT NOT NULL,
    room_number VARCHAR(20) NOT NULL,
    checkin_date DATE,
    checkout_date DATE,
    FOREIGN KEY (attendee_id) REFERENCES attendees(id) ON DELETE CASCADE
) ENGINE=InnoDB;
```

## 5. 设计说明

### 5.1 为什么用 ENUM 而不是 VARCHAR？

`role` 和 `status` 类字段只有固定几个取值，使用 ENUM 可以：
- 在数据库层面约束非法值
- 比 VARCHAR 节省存储空间
- 课程要求体现数据库设计能力

### 5.2 为什么 attendees 表要有 UNIQUE(user_id, conference_id)？

防止同一用户重复加入同一会议，这是业务规则在数据库层面的兜底保障。

### 5.3 级联删除策略

所有外键均使用 `ON DELETE CASCADE`，删除会议时自动清理相关的 invite_codes、attendees 数据。但实际代码中应在 Service 层做业务判断（如已有人报名的会议不允许删除），而不是仅依赖数据库级联。

### 5.4 字符集选择 utf8mb4

MySQL 的 `utf8` 实际上是 3 字节精简版，不支持 emoji 等 4 字节字符。使用 `utf8mb4` 是标准做法。
