### 1.2 分层职责

| 层 | 职责 | 规范 |
|----|------|------|
| View (JSP) | 页面展示，表单，Ajax 调用 | 不写 Java 业务逻辑，仅用 JSTL + EL 展示数据 |
| Controller (Servlet) | 接收请求，参数校验，调用 Service，返回 JSON 或转发 JSP | 不写 SQL，不写复杂业务逻辑 |
| Service | 业务逻辑，事务协调，调用 DAO | 每个模块一个 Service 类 |
| DAO | 数据库 CRUD，返回 VO 对象 | 使用 PreparedStatement，JDBCUtil 统一获取连接 |
| VO (Entity) | 数据实体，对应数据库表 | 私有字段 + getter/setter |

### 2.2 页面加载方式

- **独立页面**：`login.jsp`、`register.jsp`、`forgetPassword.jsp`、`profile.jsp`、`join_meeting.jsp` 为独立 HTML 页面，通过浏览器地址栏直接访问
- **框架页面**：`index1.jsp`（管理员）、`index2.jsp`（用户）为 SPA 主框架，通过 `loadPage()` 函数 fetch + innerHTML 动态加载子页面片段
- **内容片段**：`attendee/` 和 `fragments/` 下的 JSP 为内容片段，被框架通过 AJAX 动态加载，不含完整 HTML 结构

## 3. 导航流程

### 3.1 登录后路由

```
                    登录成功
                       │
              ┌────────┼────────┐
              ▼        ▼        ▼
           role=    role=    role=
         普通用户  组织者   管理员(=1)
         (=0)    (=0)        │
              │        │        │
              ▼        ▼        ▼
          index2.jsp   ← 共用 →  index1.jsp
              │                     │
    （用户端菜单）          （管理员端菜单）
```

### 3.2 典型业务流程

**参会者参会流程**：
```
打开会议大厅 → 搜索/邀请码加入 → 审核通过则跳转报名页 → 
填写往返时间、住宿要求、其他要求 → 确认参加 → 虚拟缴费 → 完成
```

**组织者创建会议流程**：
```
创建会议（模态框） → 填写信息 → 提交(待审核) → 等待管理员审核 → 
审核通过 → 生成邀请码 → 查看参会名单 → 会议当天签到管理 → 分配房间
```

**管理员审核流程**：
```
登录 → 会议审核列表 → 查看会议详情 → 通过/驳回(填写原因) → 完成
```

### 3.3 权限拦截

- 未登录用户访问任何页面：Filter 返回 401 → 前端弹窗"登录已过期" → 跳转登录页
- 已登录但账号被禁用：Filter 返回 403 → 前端弹窗"账号已被禁用" → 跳转登录页
- 前端通过 `common.js` 全局 fetch 拦截器统一处理，后端通过 `DisableCheckFilter` 统一拦截
- 白名单路径：登录、注册、重置密码、静态资源（CSS/JS/图片）
