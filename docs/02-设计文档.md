## 1. 系统架构

### 1.1 整体架构

```
┌──────────────────────────────────────────────────┐
│                   浏览器 (Browser)                 │
├──────────────────────────────────────────────────┤
│              View 层 (JSP + Bootstrap 5)          │
│         login.jsp  register.jsp  index.jsp        │
│         conference_*.jspr  attend_*.jsp           │
│         payment_*.jsp  checkin_*.jsp  admin_*.jsp │
├──────────────────────────────────────────────────┤
│            Controller 层 (Servlet)                │
│   LoginServlet  RegisterServlet  LogoutServlet    │
│   ConferenceServlet  AttendeeServlet              │
│   PaymentServlet  CheckinServlet  AdminServlet    │
├──────────────────────────────────────────────────┤
│              Service 层 (业务逻辑)                 │
│   UserService  ConferenceService  AttendeeService │
│   PaymentService  CheckinService  AdminService    │
├──────────────────────────────────────────────────┤
│               DAO 层 (数据访问)                    │
│   UserDao  ConferenceDao  AttendeeDao             │
│   PaymentDao  CheckinDao  InviteCodeDao            │
├──────────────────────────────────────────────────┤
│               MySQL 8.0 数据库                     │
│   users  conferences  invite_codes  attendees     │
│   payments  checkins  accommodations             │
└──────────────────────────────────────────────────┘
```

### 1.2 分层职责

| 层 | 职责 | 规范 |
|----|------|------|
| View (JSP) | 页面展示，表单，Ajax 调用 | 不写 Java 业务逻辑，仅用 JSTL + EL 展示数据 |
| Controller (Servlet) | 接收请求，参数校验，调用 Service，返回 JSON 或转发 JSP | 不写 SQL，不写复杂业务逻辑 |
| Service | 业务逻辑，事务协调，调用 DAO | 每个模块一个 Service 类 |
| DAO | 数据库 CRUD，返回 VO 对象 | 使用 PreparedStatement，JDBCUtil 统一获取连接 |
| VO (Entity) | 数据实体，对应数据库表 | 私有字段 + getter/setter |

### 2.2 UI 布局规范

**登录/注册页**（无导航栏）：
```
┌──────────────────────────────┐
│                              │
│       ┌─────────────┐        │
│       │   登录/注册   │        │
│       │   表单卡片    │        │
│       └─────────────┘        │
│                              │
└──────────────────────────────┘
```

**后台页面**（登录后）：
```
┌──────────────────────────────────────────────┐
│  导航栏：Logo | 菜单项... | 用户名 [退出]      │
├────────┬─────────────────────────────────────┤
│ 左侧   │                                     │
│ 菜单   │         主内容区                      │
│        │    (根据角色显示不同内容)              │
│ ▪ 菜单1 │                                     │
│ ▪ 菜单2 │                                     │
│ ▪ 菜单3 │                                     │
│        │                                     │
└────────┴─────────────────────────────────────┘
```

- 导航栏使用 `navbar navbar-expand-lg navbar-dark bg-primary`
- 左侧菜单固定宽度 250px，使用 `list-group`
- 参会者菜单：会议大厅 / 我的参会 / 缴费记录
- 组织者菜单：会议管理 / 特邀码 / 签到管理 / 入住管理
- 管理员菜单：会议审核 / 用户管理 / 系统信息

### 2.3 UI 风格统一要求

| 元素 | 规范 |
|------|------|
| 主题色 | Bootstrap primary (#0d6efd) |
| 字体 | 系统默认（中文：Microsoft YaHei） |
| 卡片 | 使用 `card` + `shadow-sm` |
| 表格 | `table table-striped table-hover` |
| 按钮 | 主操作 `btn-primary`，删除 `btn-danger`，取消 `btn-secondary` |
| 表单 | `form-control` + `form-label`，必填项标红 `*` |
| 提示 | 成功 `alert-success`，错误 `alert-danger`，用 `alert-dismissible` 可关闭 |
| 模态框 | 删除确认等操作使用 Modal |

## 3. 导航流程

### 3.1 登录后路由

```
                    登录成功
                       │
              ┌────────┼────────┐
              ▼                 ▼
           role= 0           role= 1  
              │                 │
              ▼                 ▼
          index2.jsp        index1.jsp
       (参会者/组织者菜单)   (管理员菜单)
```

### 3.2 典型业务流程

**参会者参会流程**：
```
浏览会议大厅 → 点击会议查看详情 → 输入邀请码加入 → 填写到达/离开时间、住宿要求 → 模拟缴费 → 完成
```

**组织者创建会议流程**：
```
创建会议 → 填写信息 → 提交(待审核) → 等待管理员审核 → 审核通过 → 生成特邀码 → 查看参会名单 → 会议当天签到管理 → 分配房间
```

**管理员审核流程**：
```
登录 → 会议审核列表 → 查看会议详情 → 通过/驳回(填写原因) → 完成
```

## 4. 技术选型与版本

| 技术 | 版本 | 说明 |
|------|------|------|
| Java | JDK 17 或 21 | LTS 版本 |
| Servlet API | Jakarta EE 10 | `jakarta.servlet.*` |
| JSP | Jakarta EE 10 | 配合 JSTL 使用 |
| Tomcat | 10.1.x | 支持 Jakarta EE 10 |
| MySQL | 8.0 | 关系型数据库 |
| JDBC | 原生 JDBC（无 ORM） | 课程要求 |
| JSON | Jackson 2.18.x | Java 对象 ↔ JSON |
| 前端 | Bootstrap 5.3 + 原生 JS | CDN 引入 |
| 构建 | Maven 3.9+ | IDEA 内置 |
| 协作 | Git + GitHub | 分支策略见方案文档 |
| 部署 | 云服务器 `47.113.22.1` | Tomcat 部署 WAR 包 |
