# 会务管理系统 — 需求分析文档

> 版本：v1.0  
> 日期：2026-05-25  
> 小组：6人

---

## 1. 系统概述

### 1.1 项目背景

随着学术会议、行业论坛、企业年会等各类会议日益增多，传统的线下人工管理模式效率低下、信息传递不及时、数据难以追溯。会务管理系统旨在为各类会议提供一站式线上管理平台，覆盖会议创建、参会报名、缴费、签到、入住管理全流程。

### 1.2 目标用户

| 角色 | 描述 |
|------|------|
| **参会者 (Attendee)** | 注册后浏览会议、报名参会、填写行程信息、缴费 |
| **组织者 (Organizer)** | 注册后创建和管理自己组织的会议，管理签到和入住 |
| **管理员 (Admin)** | 审核会议、管理所有用户、维护系统 |

### 1.3 核心价值

- 组织者：快速创建会议，生成特邀码，管理参会人员签到入住
- 参会者：一键浏览会议、加入会议、在线缴费
- 管理员：统一审核会议，管理系统用户

---

## 2. 角色分析

### 2.1 参会者（Attendee）

```
注册/登录 → 浏览会议大厅 → 搜索会议 → 通过邀请码加入会议 → 填写参会信息 → 缴费 → 查看参会状态
```

### 2.2 组织者（Organizer）

```
注册/登录 → 创建会议 → 管理会议信息 → 生成/管理特邀码 → 查看参会名单 → 签到管理 → 入住分配
```

### 2.3 管理员（Admin）

```
登录 → 审核会议(通过/驳回) → 管理用户(查看/禁用) → 系统信息维护
```

> **注意**：管理员账号由系统初始化时预设，不开放注册。

## 3. 功能需求（按模块）

### 3.1 M1 — 用户系统（组长负责）

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 用户注册 | 填写用户名、密码、手机、邮箱 | P0 |
| 用户登录 | 验证用户名密码，角色跳转首页 | P0 |
| 退出登录 | 清除 Session，跳转到登录页 | P0 |
| 个人中心 | 查看和修改个人信息 | P1 |
| 权限拦截 | 未登录用户访问任何页面均跳转登录页；不同角色不能访问他人的功能页面 | P0 |
| 密码加密 | 密码使用 MD5 + 固定盐值加密存储，不存明文 | P0 |

**规则**：
- 同一用户名不能重复注册
- 管理员账号由系统预设，不允许通过注册页面创建
- 登录后将用户信息存入 Session

### 3.2 M2 — 会议管理（成员A负责）

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 创建会议 | 填写标题、描述、地点、开始日期、结束日期，提交后状态为"待审核" | P0 |
| 编辑会议 | 修改自己创建的会议信息（已审核通过的不可编辑） | P0 |
| 删除会议 | 删除自己创建的会议（级联删除相关数据） | P1 |
| 我的会议列表 | 查看自己创建的会议，显示审核状态 和相关信息| P0 |
| 特邀码生成 | 为某会议生成邀请码、由26个大小写字母和阿拉伯数字组成共十位| P0 |
| 特邀码管理 | 查看会议的报名人、参会者的基本信息（用户名、往返时间、要求等等）| P1 |
**规则**：
- 会议创建后默认状态为 `pending`（待审核）
- 只有状态为 `approved`（已通过）的会议才对参会者可见
- 每个会议只有审核通过后才有邀请码，且每个会议对应唯一的邀请码
- 会议过了规定时间就会变为已结束状态，无法进行任何修改（可删除该会议）
### 3.3 M3 — 参会管理（成员B负责）
| 功能 | 描述 | 优先级 |
|------|------|--------|
| 浏览会议列表 | 查看所有已审核通过的会议，展示标题、时间、地点、住宿地址 | P0 |
| 搜索会议 | 按关键词（标题）搜索会议 | P1 |
| 会议详情 | 查看单个会议的详细信息 | P0 |
| 加入会议 | 通过邀请码加入会议 | P0 |
| 我的参会 | 查看自己已报名的会议列表 | P0 |

**规则**：
- 只有 `status = 'approved'` 的会议才出现在浏览列表中
- 邀请码验证失败时给出明确提示（码不存在/已过期）
- 同一用户不能重复加入同一会议

### 3.4 M4 — 参会信息与缴费（成员C负责）

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 填写参会信息 | 加入会议前要先填写往返时间、住宿要求、其他情况 | P0 |
| 修改参会信息 | 修改已填写的参会信息 | P1 |
| 虚拟缴费 | 选择参会记录，输入金额，模拟支付 | P0 |
| 缴费记录 | 查看自己的缴费记录和缴费状态 | P0 |

**规则**：
- 必须先加入会议（有 attendees 记录）才能缴费
- 每笔缴费关联一条参会记录
- 同一参会记录只允许一次缴费
- 参会的申请信息和参会信息的修改都要通知组织者
### 3.5 M5 — 签到与入住管理（成员D负责）
| 功能 | 描述 | 优先级 |
|------|------|--------|
| 参会者签到 | 组织者为参会者手动签到（按姓名搜索参会者列表） | P0 |
| 签到统计 | 显示某会议的已签到/未签到人数及名单 | P0 |
| 房间分配 | 为已签到的参会者分配房间号 | P1 |
| 入住管理 | 记录入住日期、退房日期，标记退房(0未退房\1代表已退房)| P1 |
**规则**：
- 签到操作由组织者执行（checked_by 记录操作人）
- 同一参会者同一会议只能签到一次
- 只有签到后才可以分配房间

### 3.6 M6 — 管理员面板（成员E负责）

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 会议审核 | 查看所有待审核的会议，通过或驳回 | P0 |
| 会议列表 | 查看所有会议（含各状态统计） | P0 |
| 用户管理 | 查看所有用户列表，启用/禁用用户账号 | P0 |
| 系统信息 | 查看系统基本统计（会议数、用户数、参会数等） | P1 |
**规则**：
- 只有 `role = 'admin'` 的用户能访问管理员面板
- 被禁用的用户无法登录
- 驳回会议时可填写驳回原因

## 4. 非功能需求
| 需求 | 描述 |
|------|------|
| 页面风格 | 统一使用 Bootstrap 5 组件，全站风格一致（配色、字体、间距） |
| 操作反馈 | 所有增删改操作需有成功/失败提示（Alert 或 Toast） |
| 安全性 | 密码加密存储（MD5 + 盐值）；Session 过期时间为 30 分钟 |
| 兼容性 | 支持 Chrome、Edge 浏览器 |
| 响应式 | 基于 Bootstrap Grid，适配 14 寸笔记本屏幕即可 |
| 数据校验 | 前端 + 后端双重校验，防止空值和非法数据提交 |
