# 会务管理系统 — 架构设计

---

## 1. 整体架构

本项目采用经典的 **View-Controller-Service-DAO 四层分层架构**，并在前端引入 Filter 作为横切拦截层，遵循"高内聚、低耦合"的设计原则。每层职责清晰，上层依赖下层，下层感知不到上层。

```
┌─────────────────────────────────────────────────┐
│                  View 层 (JSP + CSS + JS)        │
│   独立页面：login.jsp  register.jsp  join_meeting.jsp │
│   框架页面：index1.jsp（管理员） index2.jsp（用户）   │
│   内容片段：attendee/*.jsp  fragments/*.jsp        │
├─────────────────────────────────────────────────┤
│               Filter 层 (横切拦截)                │
│   DisableCheckFilter  登录验证 + 账号禁用拦截      │
├─────────────────────────────────────────────────┤
│             Controller 层 (Servlet)              │
│   UserServlet  ConferenceServlet  JoinServlet    │
│   SearchServlet  ConferenceListServlet           │
│   PaymentServlet  CheckinServlet  AdminController │
├─────────────────────────────────────────────────┤
│              Service 层 (业务逻辑)                │
│   UserService  ConferenceService  AttendeeService│
│   PaymentService  CheckinService                 │
│   AccommodationService  StatisticsService         │
├─────────────────────────────────────────────────┤
│               DAO 层 (数据访问)                   │
│   接口：UserDao  ConferenceDao  AttendeeDao      │
│         PaymentDao  CheckinDao  AccommodationDao │
│   实现：*Impl  +  JDBCUtil（数据库连接工具）       │
├─────────────────────────────────────────────────┤
│              VO 层 (实体类)                       │
│   User  Conference  Attendee  Payment            │
│   Checkin  Accommodation                        │
├─────────────────────────────────────────────────┤
│              MySQL 8.0 数据库                     │
│   users  conferences  attendees                  │
│   payments  checkins  accommodations             │
└─────────────────────────────────────────────────┘
```

---

## 2. 设计原则

| 原则 | 在本项目中的体现 |
|------|-----------------|
| 单一职责原则 | 每个类只负责一个功能领域：UserServlet 只管用户相关操作，ConferenceServlet 只管会议 CRUD |
| 接口隔离原则 | DAO 层定义接口（如 UserDao），实现类（UserDaoImpl）分离，调用方只依赖接口 |
| 依赖倒置原则 | Service 层持有 Dao 接口引用，不依赖具体实现类，方便替换 |
| 开闭原则 | 新增功能模块只需新建 Servlet + Service + Dao，不影响已有代码 |

---

## 3. 各层职责与规范

| 层 | 职责 | 规范 |
|----|------|------|
| View (JSP) | 页面展示、表单交互、AJAX 调用 | 不写 Java 业务逻辑，纯 JSTL + EL + JS；独立页面完整 HTML，框架片段不含 `<html>` 结构 |
| Filter | 登录验证、账号禁用拦截 | 不写业务逻辑，只做放行或拦截；AJAX 返回 JSON（401/403），页面请求重定向至 login.jsp；白名单集中管理 |
| Controller (Servlet) | 接收请求、参数校验、调用 Service、返回 JSON | 不写 SQL，不写复杂业务逻辑；统一返回 `{code, msg, data}` 格式 |
| Service | 业务逻辑编排、事务协调、调用 DAO | 每个模块一个 Service 类，组合调用多个 DAO |
| DAO | 数据库 CRUD，返回 VO 对象 | 使用 PreparedStatement 防注入；JDBCUtil 统一获取连接；接口与实现分离 |
| VO (Entity) | 数据实体，对应数据库表 | 私有字段 + getter/setter，字段名与数据库列对应 |

## 5. 前端页面组织

按角色和功能模块组织，分为独立页面、框架页面和内容片段三种：

| 类型 | 文件 | 说明 |
|------|------|------|
| 独立页面 | `login.jsp` `register.jsp` `forgetPassword.jsp` | 完整 HTML，无框架，直接访问 |
| 独立页面 | `join_meeting.jsp` | 参会登记页，独立 HTML |
| 独立页面 | `profile.jsp` | 个人中心 |
| 框架页面 | `index1.jsp` | 管理员主框架，侧边栏 + 动态内容区 |
| 框架页面 | `index2.jsp` | 用户主框架，侧边栏 + 动态内容区 |
| 内容片段 | `attendee/default.jsp` | 用户首页 |
| 内容片段 | `attendee/conferenceHall.jsp` | 会议大厅（卡片式浏览 + 邀请码加入） |
| 内容片段 | `attendee/meetingSearch.jsp` | 会议搜索（表格式） |
| 内容片段 | `attendee/myAttendances.jsp` | 我的参会记录（筛选 + 分页） |
| 内容片段 | `attendee/myConferences.jsp` | 我的会议（列表 + 创建/编辑模态框） |
| 内容片段 | `attendee/conferencePayment.jsp` | 缴费记录（统计卡片 + 明细表格） |
| 内容片段 | `checkin_manage.jsp` `room_manage.jsp` | 签到与入住管理 |
| 内容片段 | `fragments/overview.jsp` | 管理员仪表盘概览 |
| 内容片段 | `fragments/conferences.jsp` | 管理员会议审核列表 |
| 内容片段 | `fragments/users.jsp` | 管理员用户管理列表 |
| 内容片段 | `fragments/stats.jsp` | 管理员系统统计页 |
| 内容片段 | `admin_conferences.jsp` `admin_users.jsp` `admin_stats.jsp` | 管理员后台页面 |
| 内容片段 | `admin_conferences_fragment.jsp` `admin_users_fragment.jsp` `admin_stats_fragment.jsp` | 管理员后台内容片段 |
| 静态资源 | `css/*.css` `js/*.js` | 各页面独立 CSS/JS 文件 |

**前端交互模式**：`index2.js` 的 `loadPage(pageName)` 通过 Fetch API 异步加载 JSP 片段并注入 `#pageContent`，加载后重建 `<script>` 标签使其执行，实现页面无刷新切换。

**全局拦截器**：`common.js` 劫持 `window.fetch`，自动给所有请求加 `X-Requested-With` 头，捕获 401/403 响应弹窗提示并跳转登录页。

## 6. Servlet 路由设计

| Servlet | URL 映射 | 主要方法 |
|---------|---------|---------|
| UserServlet | `/user/*` | `/login` `/register` `/profile` `/logout` `/update` `/changePassword` `/resetPassword` |
| ConferenceServlet | `/conference/*` | `/create` `/update` `/delete` `/detail` `/genCode` `/myList` |
| SearchServlet | `/conference/search` | POST：关键字/邀请码搜索 |
| JoinServlet | `/attendee/join` | POST：创建参会记录 |
| CheckStatusServlet | `/attendee/checkStatus` | POST：检查是否已报名 |
| ConferenceListServlet | `/attendee/myList` | POST：获取我的参会列表（关联会议+缴费状态） |
| CancelAttendee | `/attendee/cancel` | POST：取消参会 |
| PaymentServlet | `/payment/*` | `/create` `/pay` `/history` `/delete` |
| AdminController | `/admin/*` | 会议审核、用户管理 |

**路由风格**：统一使用 `@WebServlet("/xxx/*")` 注解，在 `doGet`/`doPost` 中通过 `request.getPathInfo()` 获取子路径，用 `switch` 分发到对应的私有处理方法。

## 7. 统一 JSON 响应格式

所有后端接口统一返回以下结构：

```json
{
    "code": 200,
    "msg": "操作成功",
    "data": { ... }
}
```

| code | 含义 |
|------|------|
| 200 | 操作成功 |
| 300 | 已报名（参会状态检测专用） |
| 400 | 参数错误 / 业务逻辑拒绝 |
| 401 | 未登录 |
| 403 | 账号已被禁用 |
| 500 | 服务器内部错误 |

---

## 8. 关键设计决策

- **接口与实现分离**：DAO 层全部采用接口 + Impl 模式，便于测试和扩展
- **Servlet 统一入口**：每个 Servlet 通过 `@WebServlet("/xxx/*")` + `getPathInfo()` 路由到多个处理方法，避免 Servlet 类膨胀
- **统一 JSON 响应**：所有接口返回 `{code, msg, data}` 结构，前端统一解析
- **JSP 不嵌 Java 逻辑**：业务逻辑全部在 JS 和后端 Java，JSP 仅用 JSTL + EL 展示数据
- **SPA 体验**：通过 `loadPage()` 异步加载和脚本重执行实现页面内无刷新切换
- **全局安全拦截**：Filter 负责后端统一拦截，`common.js` 负责前端统一弹窗，双层防护
- **`X-Requested-With` 头标记**：`common.js` 自动注入，Filter 据此区分 AJAX 与页面请求，返回 JSON 或重定向
